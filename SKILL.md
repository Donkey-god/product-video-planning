# 生鲜商品视频策划技能

## 固定配置

```
视频输出根目录：~/freshVedioPlanning/
视频规格：时长 ≤15s | 分辨率 720p | 比例 9:16
即梦CLI命令：~/.local/bin/dreamina image2video
即梦提示词字数限制：≤1900字（按UTF-8字符计）
```

## 触发条件

每次任务开始前，**逐项收集商品信息**（见下方清单），如有缺失立即暂停并请求用户提供。不得跳过，不得自行搜索或推理补充。

## 第一步：收集商品信息

向用户收集以下**全部**字段，缺一不可：

| 字段 | 说明 | 示例 |
|------|------|------|
| `product_name` | 商品名称 | 美汁源益生元凤梨百香果复合果汁饮料 |
| `sku` | 商品SKU编号 | 556127 |
| `selling_points` | 商品核心卖点（多条，用换行分隔） | 30%膳食纤维、凤梨+百香果双果、0脂 |
| `user_demands` | 用户需求/使用场景（多条） | 餐后解腻、肠道健康、下午茶 |
| `hot_topics` | 当前热点/季节机会（可选填） | 轻体季、健康饮品 |
| `main_image` | 商品主图本地路径 | ~/freshVedioPlanning/.../main_image.jpg |
| `detail_images` | 商详图片路径列表（至少1张） | ~/freshVedioPlanning/.../detail1.jpg |

**缺少任意字段 → 立即告知用户缺失项，停止后续流程。**

## 第三步：切入点分析

将分析结果保存至会话目录的 `01_analysis.md`。

必须输出以下结构，不得省略：

### 3.1 人群-场景-情绪三联分析

围绕商品卖点，先给出3组不同的「人群×场景×情绪触发」组合，每组包含：

- 目标人群（如：久坐白领、健身人群、宝妈、学生）
- 真实场景（如：工位犯困、餐后负担、通勤闷热、下午茶社交）
- 情绪触发（如：惊喜、解压、治愈、身份认同、社交炫耀）
- 对应消费动机（功能性/情绪性/社交性）

### 3.2 爆点钩子设计（前3秒）

至少产出3个差异化开场钩子，并标注所属类型：

- 视觉奇观型（巨物化、反重力、材质异化、色彩冲击）
- 叙事反转型（“以为A其实B”、错位空间、时间冻结）
- 关系代入型（第一视角、对话感、可模仿动作）

每个钩子都要写明「画面动作 + 一句话文案 + 为什么能停留」。

### 3.3 卖点转译策略（禁止直白罗列）

将理性卖点转为可视化语言，至少覆盖：

- 成分/功能如何“看得见”（液体状态、气泡、光线、果实动态）
- 口感如何“听得见”（开瓶声、倒液声、气泡声、咽下反馈）
- 品牌记忆点如何“记得住”（颜色符号、固定动作、收尾口号）

### 3.4 平台适配约束（抖音/即梦）

在分析末尾给出“本次必须遵守”的4条硬约束：

1. 前1秒必须出现视觉主体（果实/瓶身/动作）
2. 前3秒必须出现冲突或反差（尺度、空间、情绪）
3. 每4秒至少一次信息增量（新动作/新角度/新卖点）
4. 结尾2秒必须完成品牌落版+记忆锚点（标签/口号/动作）

```

核心反差：
视觉母题（贯穿符号）：

  - Hook（0-3秒）：
  - Build（3-10秒）：
  - Payoff（10-15秒）：

  - 至少1个“空间/尺度/时间”错位设计
  - 至少1个“可模仿动作”用于社交传播

  - 可能翻车点：
  - 规避策略：
适配商品卖点：

```

### 第四步额外要求（提高创新密度）

- 三套创意必须分别覆盖三种方向：  
  A. 视觉奇观型（如“巨物化果实”“材质魔法”）  
  B. 情绪治愈型（如“自然手作”“日常仪式感”）  
  C. 社交话题型（如“工位梗”“反差挑战”）
- 每套需给出1句可做封面的短句（12字以内）
- 每套需给出1个可延展的话题标签方向（如：#工位仪式感）

## 第五步：生成分镜脚本（三套 + 打分）

生成3套完整分镜脚本，保存至 `03_scripts.md`。

### 5.1 脚本生成原则（用于提交即梦前）

必须遵循：

1. **内容优先**：每镜必须有“可被拍出来的动作变化”，禁止空泛形容词堆砌。  
2. **创新优先**：每套至少包含1个反常规镜头机制（错位、反转、拟物、时间切片）。  
3. **信息节奏**：平均每2-3秒出现新信息点（新动作/新卖点/新视角）。  
4. **品牌可记忆**：颜色锚点、道具锚点、动作锚点至少命中2项。  
5. **即梦可执行**：描述具体主体、动作、场景、光线、景别，避免抽象命令。

### 5.2 分镜脚本统一格式（升级版，严格执行）

### 5.3 三套脚本评分与推荐（必须量化）

三套全部生成后，必须进行量化评分并展示排名。评分表保存于 `03_scripts.md` 末尾。

评分维度（总分100）：

- 开场停留力（20分）：前3秒是否有强钩子
- 创意新鲜度（20分）：是否具备反常规机制
- 卖点可感知度（20分）：功能是否被“看见/听见”
- 节奏完成度（15分）：15秒内是否完成起承转合
- 品牌记忆度（15分）：是否形成视觉/动作锚点
- 即梦执行稳定性（10分）：提示词是否清晰可生成

| A | 18 | 17 | 16 | 13 | 14 | 9 | 87 |
| B | ... | ... | ... | ... | ... | ... | ... |
| C | ... | ... | ... | ... | ... | ... | ... |

## 第六步：视频生成（调用即梦CLI图生视频）

### 6.1 构造即梦image2video提示词

#### 6.1.1 使用“四段式高保真组装器”生成即梦提示词

**目标**：最大化保留第五步的创意密度，同时提升 Seedance 2.0 的可执行性与稳定性。

**⚠️ 强制规则（必须全部满足）**
- 不允许仅粘贴散乱字段，必须组装为四段式结构后再提交
- 必须保留“创意核心句 + 情绪定位 + 核心反差”三要素，禁止在压缩时丢失
- 先保证“画面动作与镜头语言”，再保留文案修饰；禁止本末倒置
- 若字数允许，优先保留完整版本；超字时按“降级优先级”精简

**四段式结构（提交即梦时的最终文本骨架）**：

1. `【整体描述】`  
   包含：风格、时长、比例、主色调、光影、人物限制、画面质感。

2. `【分镜描述】`  
   按时间顺序列出每镜（建议4-5镜），每镜至少保留：  
   - 画面内容（主体+动作+场景）  
   - 运镜手法（景别/移动/节奏）  
   - 卖点文字（主标/副标，可简写）  
   - 旁白配音（可保留关键句）  
   - 音效/环境声（关键拟音）

3. `【声音说明】`  
   包含：BGM风格、节奏变化、关键音效落点（如开瓶、落果、气泡、收尾声）。

4. `【参考素材】`  
   使用 `@图片N` 逐条注明用途（构图/配色/液体动态/落版氛围）。

**操作步骤**：
1. 从 `03_scripts.md` 读取用户选定方案的完整内容
2. 提取并标准化为上述四段式结构（字段顺序固定）
3. 在提示词末尾追加统一收尾语：  
   `淡出收尾，电影感慢动作质感，商业广告水准，高完成度短视频广告。`
4. 将最终提示词写入确认artifact，等待用户确认

#### 6.1.2 字数与保真策略（1900字内）

先统计字数；若超出1900，按以下顺序精简（从上到下逐级执行）：

1. 删除解释性括号和冗余修饰词（不删核心动作）
2. 压缩“卖点文字”的字体/字号/透明度等版式参数
3. 旁白保留关键句，删情绪注释和重复修辞
4. 参考素材由长句改为“@图片N + 用途短语”
5. 每镜仅保留“画面内容+运镜手法+关键音效+一句卖点”

**禁止删减项（底线）**：
- 开场钩子镜头（前3秒）
- 至少1个反差/反转机制
- 品牌落版镜头（结尾2秒）
- 商品主体与核心卖点映射关系

**质量自检（提交前逐项勾选）**：
- 是否仍然一眼能看出“这条片子的核心创意”
- 是否仍然保留“前3秒钩子 + 中段推进 + 结尾落版”
- 是否每镜都有可执行的动作与镜头语言
- 是否保留品牌记忆锚点（颜色/动作/道具）

### 6.2 调用即梦CLI

**关键：使用主图作为全帧参考，一次提交生成完整视频（不分镜提交）。**

```bash
~/.local/bin/dreamina image2video \
  --image "<会话目录main_image.jpg路径>" \
  --prompt "<用户确认后的即梦提示词>" \
  --duration <总秒数，4-15之间> \
  --model_version seedance2.0fast \
  --video_resolution 720p \
  --poll 60
```

**注意事项：**
- `--duration` 支持 4-15 秒（seedance2.0fast）
- 如果总时长超过15秒，分多次提交，或截取核心镜头
- 如果 `--duration` 报错"should >=4 && <=15"，改为合法值
- `--poll 60` 表示提交后等待最多60秒查结果，超时则返回排队状态，任务继续在后台排队
- 任务提交成功后会返回 `submit_id`，用于后续查询

### 6.3 查询与下载

```bash
# 查询状态
~/.local/bin/dreamina query_result --submit_id <submit_id>

# 等待完成后下载（当 gen_status=success 时）
~/.local/bin/dreamina download --submit_id <submit_id> --output_path "<run目录/full_video.mp4>"
```
