# 生鲜商品视频策划技能

## 固定配置

```
视频输出根目录：~/freshVedioPlanning/
视频规格：时长 ≤15s | 分辨率 720p | 比例 9:16
即梦CLI命令：~/.local/bin/dreamina multimodal2video（全能参考模式，支持多图）
即梦提示词字数限制：≤1900字（按UTF-8字符计）
```

## 触发条件

每次任务开始前，**逐项收集商品信息**（见下方清单），如有缺失立即暂停并请求用户提供。不得跳过，不得自行搜索或推理补充。

## 第一步：收集商品信息

向用户收集以下字段（其中 `product_analysis` 为选填，其余为必填）：

| 字段 | 说明 | 示例 |
|------|------|------|
| `product_name` | 商品名称 | 美汁源益生元凤梨百香果复合果汁饮料 |
| `sku` | 商品SKU编号 | 556127 |
| `selling_points` | 商品核心卖点（多条，用换行分隔） | 30%膳食纤维、凤梨+百香果双果、0脂 |
| `user_demands` | 用户需求/使用场景（多条） | 餐后解腻、肠道健康、下午茶 |
| `hot_topics` | 当前热点/季节机会（可选填） | 轻体季、健康饮品 |
| `product_analysis` | 商品分析（选填，可粘贴类目表现/搜索趋势/行业调研/价格力/竞争力等结论） | 细分类目表现、搜索热度、行业分析、价格力、竞争力、总分 |
| `main_image` | 商品主图本地路径 | ~/freshVedioPlanning/.../main_image.jpg |
| `detail_images` | 商详图片路径列表（至少1张） | ~/freshVedioPlanning/.../detail1.jpg |

**缺少任意必填字段 → 立即告知用户缺失项，停止后续流程。`product_analysis` 为选填，不作为阻断项。**

## 第三步：切入点分析

将分析结果保存至会话目录的 `01_analysis.md`。

若提供了 `product_analysis`，必须先提取其中可直接用于创意与卖点表达的“机会信号”，例如：
- 类目与竞争态势（差异化空间、同质化风险）
- 搜索热度与季节趋势（可借势节点、关键词）
- 价格力与人群带宽（主打性价比/功能价值/口味新鲜感）
- 行业功能认知（如益生元、膳食纤维）可作为传播话术优先级依据

仅允许引用 `product_analysis` 中有明确依据的结论，禁止杜撰未提供的数据。

必须输出以下结构，不得省略：

### 3.1 人群-场景-情绪三联分析

围绕商品卖点，先给出3组不同的「人群×场景×情绪触发」组合，每组包含：

- 目标人群（如：久坐白领、健身人群、宝妈、学生）
- 真实场景（如：工位犯困、餐后负担、通勤闷热、下午茶社交）
- 情绪触发（如：惊喜、解压、治愈、身份认同、社交炫耀）
- 对应消费动机（功能性/情绪性/社交性）

### 3.2 爆点钩子设计（前3秒）

至少产出3个差异化开场钩子，并标注所属类型：

- 视觉奇观型（巨物化、反重力、材质异化、色彩冲击）
- 叙事反转型（“以为A其实B”、错位空间、时间冻结）
- 关系代入型（第一视角、对话感、可模仿动作）

每个钩子都要写明「画面动作 + 一句话文案 + 为什么能停留」。

### 3.3 卖点转译策略（禁止直白罗列）

将理性卖点转为可视化语言，至少覆盖：

- 成分/功能如何“看得见”（液体状态、气泡、光线、果实动态）
- 口感如何“听得见”（开瓶声、倒液声、气泡声、咽下反馈）
- 品牌记忆点如何“记得住”（颜色符号、固定动作、收尾口号）

新增“真实卖点提取”步骤（必须执行）：

1. 先从主图、商详图、卖点文案中提取“可被证据支撑”的卖点清单（禁止凭空补充）。
2. 为每个卖点标注类型：  
   - A 类：产品外观/包装可直接验证  
   - B 类：原材料特性可直接验证  
   - C 类：功能或口感主张（需谨慎表达，不可伪造内部状态）
3. 只允许将 A/B 类卖点直接转成镜头；C 类卖点必须通过“场景结果、情绪反馈、抽象质感”间接表达。
4. 若最终核心卖点判定为“原材料特性”，分镜前两镜必须直接表现原材料特点，不得将原材料形态与产品形态融合（例如把原料直接表现为杯内/包装内内容）。

### 3.4 平台适配约束（抖音/即梦）

在分析末尾给出“本次必须遵守”的4条硬约束：

1. 前1秒必须出现视觉主体（果实/瓶身/动作）
2. 前3秒必须出现冲突或反差（尺度、空间、情绪）
3. 每4秒至少一次信息增量（新动作/新角度/新卖点）
4. 结尾2秒必须完成品牌落版+记忆锚点（标签/口号/动作）

## 第四步：创意结构扩展

扩展3套不同的创意结构，保存至 `02_creative_structure.md`。

若存在 `product_analysis`，每套创意需补充“分析依据映射”：说明该创意引用了哪些分析结论（如季节趋势、竞品空白、价格优势、功能认知），并标注对应的传播目标（拉新/转化/差异化）。

三套结构应在**切入角度、画面基调、叙事节奏**上形成明显差异。

并强制按以下模板输出，每套必须完整：

```
创意代号：
情绪定位：（2-3个关键词）
一句话概念（High Concept）：
核心反差：
视觉母题（贯穿符号）：
叙事骨架（Hook→Build→Payoff）：
  - Hook（0-3秒）：
  - Build（3-10秒）：
  - Payoff（10-15秒）：
创新机制：
  - 至少1个“空间/尺度/时间”错位设计
  - 至少1个“可模仿动作”用于社交传播
风险点与规避：
  - 可能翻车点：
  - 规避策略：
适配商品卖点：
适配人群场景：
```

### 第四步额外要求（提高创新密度）

- 三套创意方向不设强制覆盖要求，可自由发挥；但需确保三套在切入角度与表达方式上有明显差异
- 创意方向按“主方向 + 子方向”组织，避免概念重叠：
  A. 视觉奇观主方向（强调非常规视觉体验）
     - A1 奇幻系列：魔法质变、反物理规则、超现实场景
     - A2 微观世界：细胞级/分子级/纹理级放大视角
     - A3 小人国叙事：微缩人物与产品/原料互动
     - A4 尺度错位：巨物化、缩放反差、空间折叠
  B. 情绪治愈主方向（强调日常情绪价值）
     - B1 自然手作
     - B2 仪式感慢节奏
     - B3 陪伴治愈
  C. 社交传播主方向（强调可模仿与可讨论）
     - C1 工位梗/通勤梗
     - C2 反差挑战
     - C3 互动话题模板
  D. 理性说服主方向（强调可感知的功能论证）
     - D1 功能实验
     - D2 成分可视化
     - D3 场景前后对比
  E. 叙事表达主方向（强调剧情推进与代入）
     - E1 叙事反转
     - E2 第一视角代入
     - E3 多角色短剧情
- 说明：奇幻系列、微观世界、小人国属于“视觉奇观主方向”的子方向，与视觉奇观不冲突，而是其细分表达
- 每套需给出1句可做封面的短句（12字以内）
- 每套需给出1个可延展的话题标签方向（如：#工位仪式感）

## 第五步：生成分镜脚本（三套 + 打分）

生成3套完整分镜脚本，保存至 `03_scripts.md`。

若存在 `product_analysis`，分镜中的卖点文案与镜头重点应优先对齐“高置信分析结论”（如搜索高热词、已验证差异点、价格力优势）；未被分析支持的卖点，需降级为辅助表达，不可作为核心钩子。

### 5.1 脚本生成原则（用于提交即梦前）

必须遵循：

1. **内容优先**：每镜必须有“可被拍出来的动作变化”，禁止空泛形容词堆砌。  
2. **创新优先**：每套至少包含1个反常规镜头机制（错位、反转、拟物、时间切片）。  
3. **信息节奏**：平均每2-3秒出现新信息点（新动作/新卖点/新视角）。  
4. **品牌可记忆**：颜色锚点、道具锚点、动作锚点至少命中2项。  
5. **即梦可执行**：描述具体主体、动作、场景、光线、景别，避免抽象命令。  
6. **包装/内容不确定时禁止“内部画面”**（比拆封更严格）：当无法从可靠素材证明包装/杯/碗内的真实内容时，**禁止**出现开包/开袋/拆封/倒出镜头，也**禁止**出现任何可让人联想到“已打开并看到内容”的画面，包括但不限于：包装或杯体内部、横截面/剖面、从杯口俯拍见内容、杯内/勺上可见的颗粒果粒/分层、可透视包装内部、X 光/剖切/透明材质露出内部。表现口感与质地，请改用**与本品容器无关的抽象质感**、或**可单独落地的原材料镜头**、或**外包装合规主视觉**来承接。
7. **原材料卖点优先镜头规则**：当本条视频的最终核心卖点属于“原材料特性”（如奶源、果源、谷物、发酵工艺），第1-2镜必须先拍原材料特征（产地/形态/质地/处理过程），再进入产品镜头；且禁止将原材料镜头与产品内部形态做融合或误导性拼接。

### 5.2 分镜脚本统一格式（升级版，严格执行）

### 5.3 三套脚本评分与推荐（必须量化）

三套全部生成后，必须进行量化评分并展示排名。评分表保存于 `03_scripts.md` 末尾。

评分维度（总分100）：

- 开场停留力（20分）：前3秒是否有强钩子
- 创意新鲜度（20分）：是否具备反常规机制
- 卖点可感知度（20分）：功能是否被“看见/听见”
- 节奏完成度（15分）：15秒内是否完成起承转合
- 品牌记忆度（15分）：是否形成视觉/动作锚点
- 即梦执行稳定性（10分）：提示词是否清晰可生成

输出格式示例：

|------|--------|--------|----------|------|--------|--------|------|
| A | 18 | 17 | 16 | 13 | 14 | 9 | 87 |
| B | ... | ... | ... | ... | ... | ... | ... |
| C | ... | ... | ... | ... | ... | ... | ... |

### 5.4 选型输出要求

向用户展示：

1. 三套方案总分与排名  
2. 每套一句“为什么值得投放”  
3. 推荐方案 + 推荐理由（不超过3条）

**三套脚本均生成后，向用户展示总分排名，推荐得分最高的脚本**，询问用户选择哪套进行视频生成。

## 第六步：视频生成（调用即梦CLI图生视频）

### 6.1 构造即梦image2video提示词

#### 6.1.1 使用“四段式高保真组装器”生成即梦提示词

**目标**：最大化保留第五步的创意密度，同时提升 Seedance 2.0 的可执行性与稳定性。

**⚠️ 强制规则（必须全部满足）**
- 不允许仅粘贴散乱字段，必须组装为四段式结构后再提交
- 必须保留“创意核心句 + 情绪定位 + 核心反差”三要素，禁止在压缩时丢失
- 先保证“画面动作与镜头语言”，再保留文案修饰；禁止本末倒置
- 若字数允许，优先保留完整版本；超字时按“降级优先级”精简
- 当包装/杯内/内部形态未知时，必须追加“禁内部展示约束语”（见下方模板与 `6.1.3.1`），禁止模型生成拆封、倒出、以及**任何能展示或推断产品内部/杯内/分层/颗粒/馅料/剖面**的镜头（包含透明材质透视、X 光/剖面特效等“看似没拆但看见内部”的方式）

**四段式结构（提交即梦时的最终文本骨架）**：

1. `【整体描述】`  
   包含：风格、时长、比例、主色调、光影、人物限制、画面质感。

2. `【分镜描述】`  
   按时间顺序列出每镜（建议4-5镜），每镜至少保留：  
   - 画面内容（主体+动作+场景）  
   - 运镜手法（景别/移动/节奏）  
   - 卖点文字（主标/副标，可简写）  
   - 旁白配音（可保留关键句）  
   - 音效/环境声（关键拟音）

3. `【声音说明】`  
   包含：BGM风格、节奏变化、关键音效落点（如开瓶、落果、气泡、收尾声）。

4. `【参考素材】`  
   使用 `@图片N` 逐条注明用途（构图/配色/液体动态/落版氛围）。

5. `【负向约束】`  
   明确禁止：拆封/倒出/挖取/勺取等动作；以及**内部形态展示**（杯口俯拍、杯内分层、果粒/燕麦颗粒、横截面、透视/剖面/透明容器的内部可见、与本品同杯同容器的内景等），并给出口感表达的替代安全路径（见模板）。

**操作步骤**：
1. 从 `03_scripts.md` 读取用户选定方案的完整内容
2. 提取并标准化为上述四段式结构（字段顺序固定）
3. 若存在“包装/杯内内容不确定”情况，插入禁内部展示约束语，并将相关镜头替换为**与本品包装无关的抽象质地**、或**独立食材/工艺镜头**、或**外包装/瓶身/标签主视觉**（避免同杯/同碗承载未证实内容）
4. 在提示词末尾追加统一收尾语：  
   `淡出收尾，电影感慢动作质感，商业广告水准，高完成度短视频广告。`
5. 将最终提示词写入确认artifact，等待用户确认

#### 6.1.2 字数与保真策略（1900字内）

先统计字数；若超出1900，按以下顺序精简（从上到下逐级执行）：

1. 删除解释性括号和冗余修饰词（不删核心动作）
2. 压缩“卖点文字”的字体/字号/透明度等版式参数
3. 旁白保留关键句，删情绪注释和重复修辞
4. 参考素材由长句改为“@图片N + 用途短语”
5. 每镜仅保留“画面内容+运镜手法+关键音效+一句卖点”

**禁止删减项（底线）**：
- 开场钩子镜头（前3秒）
- 至少1个反差/反转机制
- 品牌落版镜头（结尾2秒）
- 商品主体与核心卖点映射关系

**质量自检（提交前逐项勾选）**：
- 是否仍然一眼能看出“这条片子的核心创意”
- 是否仍然保留“前3秒钩子 + 中段推进 + 结尾落版”
- 是否每镜都有可执行的动作与镜头语言
- 是否保留品牌记忆锚点（颜色/动作/道具）

#### 6.1.3.1 酸奶产品专用镜头替换库（防拆封违规）

当商品是酸奶/乳制品，且无法确认包装内状态时，优先套用以下替换规则：

| 高风险镜头（避免） | 安全替代镜头（推荐） | 可承接卖点 |
|------|------|------|
| 撕开杯盖、掀膜、开袋 | 冷藏柜取杯 + 杯身水珠微距 + 标签特写 | 新鲜冷藏、品质感 |
| 倒出入口可见内容/倒出“本品酸奶形态”到碗里 | 与本品**不同容器/无标识**的抽象乳白流体质感（丝绸/慢动作流体）+ 独立原料切镜叠化 | 丝滑口感、浓醇顺滑（不指认杯内真实样子） |
| 勺子挖取、勺上挂料、可见杯内颗粒 | 独立原料特写（草莓切面/整颗蓝莓/整粒燕麦）+ 配料与包装**分镜分场**同框（不展示杯内） | 真材实料、配料感 |
| 展示夹心/分层/杯内燕麦堆 | 在**独立小器皿**中做“抽象分层/颗粒漂浮”的示意，明确不是本品打开后的内景；或用插画/色块/粒子特效表达层次 | 层次口感、风味组合（不伪造内部） |
| 从杯口俯拍看见内容 | 封闭杯体侧面轮廓 + 外壁冷凝水珠/光影扫过/标签强对比 | 清爽、清凉、想喝感 |
| 打开后直接入口特写 | 用吸管插孔喝（不见杯内）/唇边接触杯沿（不见内容）+ 情绪表演 | 便携即饮、情绪治愈 |
| 未知内容倒入麦片/水果碗 | 仅展示食材堆叠的早餐场景，酸奶包装作为道具同框（不展示倒出内容/混合状态） | 早餐搭配、场景适配 |

酸奶类建议追加到 `【负向约束】` 的专用句：
`未提供可核验的“本品打开后内景”证据时，禁止出现杯内/口沿俯拍/勺取挂料/分层颗粒等任何内景与内部质地展示；只保留封闭外立面的制品展示 + 完全独立的原料/抽象质地镜头。`

#### 6.1.3.2 原材料卖点双镜头模板（可直接套用）

当最终核心卖点属于“原材料特性”时，优先使用以下双镜头开场模板（第1镜 + 第2镜）：

**模板A：酸奶/乳制品（奶源或发酵特性）**
- 第1镜（0-1.5s，原料识别）：牧场晨光/奶源环境 + 原奶相关元素微距（奶滴、冷凝、白色质感）
- 第2镜（1.5-3s，原料质地/处理）：发酵工艺符号化表现（乳白流体慢动作、发酵气泡抽象动画、温控工艺场景）
- 禁止：将上述乳白质感直接表现为“本品杯内状态”

**模板B：果汁/果乳饮（果源特性）**
- 第1镜（0-1.5s，原料识别）：果园/产地氛围 + 完整果实高饱和特写（带水珠、切面纹理）
- 第2镜（1.5-3s，原料质地/处理）：果肉纤维、榨取路径、风味层次的抽象化镜头（不进入产品容器内部）
- 禁止：把果肉/纤维直接合成到“产品包装内部可见形态”

**模板C：谷物饮/坚果饮（谷物坚果特性）**
- 第1镜（0-1.5s，原料识别）：燕麦/坚果独立落镜、烘焙或研磨前后对比
- 第2镜（1.5-3s，原料质地/处理）：谷物膨化纹理、研磨颗粒、香气可视化粒子
- 禁止：出现勺取挂料、杯内颗粒堆叠、透明杯内分层等“内部可见”镜头

**统一落地要求（必须满足）**
- 前两镜只承担“原材料卖点建立”，不与产品内部形态融合
- 第三镜开始再引入产品外包装/品牌符号，完成“原料 -> 产品价值”的叙事过渡
- 若素材无法支撑原材料卖点，宁可降级为外包装与场景卖点，也不要伪造内部状态

#### 6.1.4 提交前必须确认

**⚠️ 严格流程：生成提示词 → 写入手工确认artifact → 展示给用户 → 用户确认后才提交即梦CLI。禁止未经用户确认直接调用即梦。**

每次提交前，向用户展示：
1. 本次即梦提示词（完整中文，可直接复制到即梦官网测试）
2. 字数统计结果
3. 预计积分消耗
4. 参考图方案（主图必选 + 是否追加商详图 + 选择理由 + 每张图的用途说明）
5. 本次“创意保真检查”结果（钩子/反转/落版是否保留）

**参考图选择原则：**
- 主图（main_image.jpg）必须入选，作为全能参考的核心基底
- 商详图根据镜头动作需求追加：
  - 需要展示液体动态/通透感 → 优先用**不暴露本品杯内/包装内**的图（外壁冷凝、标签光影、非本品容器的牛奶/水质感示意）；**只有**商详图已明确展示合规“液体动态且来源可信、且不暗示未知内部”时才追加
  - 需要展示饮用场景/手持角度 → 优先选择**不展示口沿内景/不俯拍杯内**的持杯图（侧面持杯/手部遮挡杯口/吸管入口但不透视内部）
  - 需要展示成分/果实细节 → 追加**独立水果/原料**特写（与打开后的杯内无强绑定），避免用“勺上挂料/杯里可见颗粒”类暗示内部
- 总图片数建议 2-4 张（主图 + 1-3张精选商详图），不超过9张

用户回复"可以/好/确认"后，才执行 `dreamina multimodal2video` 命令。

### 6.2 调用即梦CLI（全能参考模式）

**关键：使用 `multimodal2video` 命令启用“全能参考”模式，支持上传多张图片作为参考。主图必须入选，商详图按需追加。一次提交生成完整视频（不分镜提交）。**

```bash
~/.local/bin/dreamina multimodal2video \
  --image "<会话目录main_image.jpg路径>" \
  --image "<商详图1路径>" \
  --image "<商详图2路径>" \
  --prompt "<用户确认后的即梦提示词>" \
  --duration <总秒数，4-15之间> \
  --ratio 9:16 \
  --model_version seedance2.0fast \
  --video_resolution 720p \
  --poll 60
```

**注意事项：**
- **必须使用 `multimodal2video` 而非 `image2video`**，前者支持全能参考和多图上传，后者仅支持单张首帧图
- `--image` 参数可重复多次，最多支持9张图片（主图必选 + 商详图按需追加）
- 主图必须入选参考图；商详图是否入选需基于“镜头动作是否需要补充细节/材质/功能展示”判断，不做固定限制
- 在提交前，先完成“主图+商详图内容分析”，再结合最终运镜脚本确定参考图清单
- `--duration` 支持 4-15 秒（seedance2.0fast）
- `--ratio` 必须显式指定为 `9:16`（竖屏），multimodal2video 不会自动推断比例
- 如果总时长超过15秒，分多次提交，或截取核心镜头
- 如果 `--duration` 报错"should >=4 && <=15"，改为合法值
- `--poll 60` 仅用于“快速拿到首个状态”，不用于长时间持续等待
- 任务提交成功后会返回 `submit_id`，必须写入会话目录的 `dreamina_task.json` 以便后续人工下载

### 6.3 长队列处理与任务记录（替代持续轮询）

高峰期视频生成可能超过2小时，禁止长时间自动轮询。

**执行策略（严格按顺序）：**
1. 提交 `multimodal2video` 后，最多进行一次短轮询（`--poll 60`）获取初始状态。
2. 若状态为排队中/处理中，且提示等待时间较长（例如预计超过30分钟或明显处于高峰队列），立即停止自动等待。
3. 将任务信息写入 `dreamina_task.json`，至少包含：
   - `submit_id`
   - `status`（queueing/processing/success/failed）
   - `created_at`
   - `last_checked_at`
   - `note`（如“高峰期长队列，改为人工触发下载”）
4. 向用户明确告知：任务已成功提交并记录，可稍后人工触发查询/下载。
5. 本轮流程到此结束，不再循环 `query_result`。

`dreamina_task.json` 示例：
```json
{
  "submit_id": "xxxxxxxx",
  "status": "queueing",
  "created_at": "2026-04-20T10:00:00+08:00",
  "last_checked_at": "2026-04-20T10:01:00+08:00",
  "note": "高峰期长队列，改为人工触发下载"
}
```

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 商品信息字段缺失 | 停止流程，列出缺失字段，请用户补充 |
| 即梦提示词字数超1900字 | 继续精简每镜描述，删除情绪标注和说明性文字 |
| 即梦CLI不可用 | 提示用户手动在官网生成，提供提示词，询问文件路径 |
| 即梦CLI执行失败 | 记录错误信息，请用户确认CLI状态，尝试回退手动模式 |
| 即梦返回长队列/预计等待过长 | 立即停止自动轮询，写入 `dreamina_task.json`，通知用户后续人工触发下载 |
| duration报错 | 调整 `--duration` 在 4-15 秒合法范围内重试 |
| 视频文件未找到 | 询问用户视频文件位置，尝试常见下载目录 |
| 目录已满/写入失败 | 告知用户，检查目录权限 |
